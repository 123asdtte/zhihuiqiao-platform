### 二、今日工作内容



#### 2.1 前端页面原型设计



完成8个核心页面的布局与交互设计：



- **首页**：顶部导航菜单（首页/科研撮合/资源流转/教学辅助/个人中心）、轮播Banner展示平台核心价值、四个快捷入口（学生/教师/企业/管理员）、数据看板（项目数/资源数/用户数/匹配成功数）、最新动态滚动列表
- **科研项目列表页**：按领域/状态筛选、关键词搜索、项目卡片（标题/负责人/团队人数/状态标签）
- **项目详情页**：完整项目信息、教师可编辑/关闭项目、学生可点击“申请加入”并填写申请动机
- **企业需求列表页**：技术需求展示、企业发布需求入口
- **资源列表页**：按分类/状态筛选、资源卡片（名称/分类/位置/价格/状态）
- **资源详情与预约页**：完整资源信息、日历组件选时间段、预约提交、拥有者审批
- **学习资源列表页**：按类型/分类筛选、资源卡片（标题/类型/上传人/浏览下载量）
- **学习中心页**：学习记录、进度、收藏资源，支持继续学习
- **登录/注册/个人主页**：用户认证与信息管理

#### 2.2 业务流程梳理



完成三大核心模块的业务流程设计：



**科研撮合闭环**

学生/教师注册 → 完善科研画像 → 教师发布项目/企业发布需求 → 学生浏览项目列表 → 查看项目详情 → 提交加入申请 → 教师审核申请 → 通过/拒绝 → 团队组建成功 → 项目启动



**资源流转闭环**

用户发布闲置资源 → 填写资源信息（名称/分类/描述/价格） → 平台审核上架 → 其他用户检索资源 → 查看详情 → 提交预约 → 资源拥有者审批 → 通过 → 线下交接使用 → 归还 → 流转记录归档



**产教融合闭环**

产业需求 → 科研课题 → 教学项目 → 人才培养 → 产业反馈





#### 2.3 用户角色与权限设计



完成四类用户角色的功能权限划分：



**学生**

- 浏览科研项目列表，查看项目详情
- 申请加入科研项目
- 浏览闲置资源，预约使用资源
- 查看个人申请记录与预约记录
- 浏览学习资源，在线学习并记录进度

**教师**

- 发布科研项目，管理项目成员
- 审核学生加入申请
- 发布闲置资源，审批资源预约
- 上传教学资源
- 查看科研项目数据统计

**企业**

- 发布技术需求
- 浏览科研项目与人才
- 查看需求匹配进度
- 企业信息管理

**管理员**

- 用户管理（审核/禁用/角色分配）
- 内容审核（项目/资源/需求审核）
- 系统数据看板
- 平台配置管理
