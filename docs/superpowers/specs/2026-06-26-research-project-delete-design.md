# 科研项目删除功能设计文档

## 1. 背景与目标

当前系统支持教师/企业用户发布科研项目，但不支持删除已发布的项目。随着测试数据和无效项目的积累，发布者需要一种安全的方式来管理自己发布的项目。同时，管理员也需要能够清理违规或过期项目。

本功能的目标：
- 允许项目发布者删除自己发布的科研项目
- 允许管理员删除任何科研项目
- 防止误删影响已申请/已加入成员的项目
- 提供清晰的前端管理入口

## 2. 设计范围

### 2.1 包含内容
- 后端删除接口及权限、状态、关联数据校验
- 前端「我的项目」管理页面
- 项目详情页删除入口
- 路由、菜单和权限配置

### 2.2 不包含内容
- 企业需求删除功能（本次仅针对科研项目）
- 学习资源、闲置资源的删除功能扩展
- 项目删除后的数据恢复功能

## 3. 后端设计

### 3.1 接口定义

```
DELETE /api/research/project/{id}
```

**请求参数：**
| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | Long | 项目 ID（路径参数） |

**响应：**
```json
{
  "code": 200,
  "message": "删除成功",
  "data": true
}
```

### 3.2 删除规则

1. **权限校验**
   - 当前用户必须是项目发布者，或角色为 `admin`
   - 否则返回 `403`："无权删除该项目"

2. **存在性校验**
   - 项目必须存在
   - 不存在则返回 `404`："项目不存在"

3. **状态校验**
   - 只允许删除状态为 `draft`、`pending_audit`、`recruiting` 的项目
   - 其他状态（`ongoing`、`completed`、`closed`、`rejected`）返回 `400`："当前状态的项目不允许删除"

4. **关联数据校验**
   - 查询该项目的申请记录
   - 如果存在状态不是 `rejected` 的申请记录，禁止删除
   - 返回 `400`："该项目已有申请记录，无法删除"

5. **级联删除**
   - 校验通过后，先删除该项目关联的所有 `project_application` 记录
   - 再删除 `research_project` 记录

### 3.3 实现位置

- **Controller:** `ResearchController.java` 新增 `deleteProject` 方法
- **Service:** `ResearchProjectService.java` / `ResearchProjectServiceImpl.java` 新增 `deleteProject` 方法
- **关联删除:** 使用 `projectApplicationService` 按 `project_id` 删除申请记录

## 4. 前端设计

### 4.1 新增「我的项目」页面

**路径：** `/app/research/my-projects`

**功能：**
- 展示当前登录用户发布的所有科研项目（不分状态）
- 支持按状态筛选：`全部`、`草稿`、`待审核`、`招募中`、`进行中`、`已结束`
- 每行展示：项目名称、类型、状态、创建时间、浏览量、操作按钮
- 操作按钮：
  - 「查看」：跳转项目详情
  - 「编辑」：跳转发布页并回填数据（如状态允许编辑）
  - 「删除」：弹出确认框，确认后调用删除接口

**可见性：**
- 仅对 `teacher`、`enterprise`、`admin` 角色显示菜单入口
- `student` 角色不显示

### 4.2 项目详情页删除入口

**位置：** `ProjectDetail.vue`

**规则：**
- 仅当当前用户是项目发布者，或角色为 `admin` 时显示「删除项目」按钮
- 点击后弹出二次确认对话框
- 确认后调用删除接口，成功后返回项目列表

### 4.3 删除确认弹窗

文案：
> 确定要删除项目「{projectName}」吗？删除后将无法恢复，且相关申请记录也会被清除。

按钮：取消 / 确定

## 5. 数据模型

涉及表：
- `research_project`：科研项目主表
- `project_application`：项目申请表

删除流程：
1. 开启事务
2. 查询项目并校验权限、状态、关联数据
3. 删除 `project_application` 中 `project_id = id` 的记录
4. 删除 `research_project` 中 `id = id` 的记录
5. 提交事务

## 6. 错误处理

| 场景 | HTTP 状态码 | 错误提示 |
|------|-------------|----------|
| 未登录 | 401 | 请先登录 |
| 无权限 | 403 | 无权删除该项目 |
| 项目不存在 | 404 | 项目不存在 |
| 状态不允许删除 | 400 | 当前状态的项目不允许删除 |
| 存在有效申请记录 | 400 | 该项目已有申请记录，无法删除 |
| 删除失败 | 500 | 删除失败，请稍后重试 |

## 7. 测试计划

### 7.1 后端测试
- 发布者删除自己的草稿项目：成功
- 发布者删除他人的项目：失败 403
- 学生删除项目：失败 403
- 管理员删除任何状态允许的项目：成功
- 删除 `ongoing` 状态项目：失败 400
- 删除有 pending 申请的项目：失败 400
- 删除只有 rejected 申请的项目：成功
- 删除后查询项目详情：404

### 7.2 前端测试
- 「我的项目」菜单按角色正确显示/隐藏
- 我的项目列表正确展示发布者的项目
- 状态筛选正常工作
- 删除按钮按权限正确显示/隐藏
- 删除成功后列表自动刷新
- 删除确认弹窗文案正确
